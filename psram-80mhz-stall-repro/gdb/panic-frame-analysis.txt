===== Interpretation (PROVEN) =====
- 0x3d887660 is a FIXED, VALID, in-pool PSRAM address computed by pure arithmetic
  (PSNAP_BASE 0x3d880000 + chunk 7*4096 + offset). It is NOT a dereferenced/corrupt
  pointer -- psnapChunk only walks a linear range. The reads at 0x3d887000..0x3d88765c
  succeeded; the cache-line-aligned word at 0x3d887660 is where the line-fill stalled.
- excvaddr=0 / pseudo exccause => no memory fault. The load simply never completed;
  the interrupt watchdog reset/halted the wedged core.
- This is a PSRAM read-path STALL of a valid address, not data corruption.
